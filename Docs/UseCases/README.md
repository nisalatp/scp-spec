# System Use Cases & Journeys

This section documents key end-to-end user journeys within the SCP system. These use cases span multiple components and roles, illustrating how the platform functions as a whole.

## Core Journeys

| Journey | Actors | Description |
| :--- | :--- | :--- |
| [Register Beneficiary](./register-beneficiary.md) | Field Agent, System | The process of identifying and registering a new person/household. |
| [Launch New Program](./launch-program.md) | Program Manager, Vertical Owner, Finance Manager | Designing, approving, and activating a new social protection program. |
| [Process Case Intervention](./process-case.md) | Field Agent, Case Worker | Identifying a need, creating a case, and resolving it. |
| [Disburse Benefits](./disburse-benefits.md) | System, Finance Manager, Payment Provider | Calculating eligibility and transferring funds/benefits. |
| [Onboard Partner](./onboard-partner.md) | Tenant Admin, Integration Manager | Setting up API access and webhooks for an external agency. |

## Journey Structure
Each Use Case document covers:
1.  **Goal**: What is being achieved.
2.  **Actors**: Who is involved.
3.  **Preconditions**: What must be true before starting.
4.  **Step-by-Step Flow**: The sequence of actions.
5.  **Component Interactions**: Which system parts are touched.
