# SCP-SPEC-OPS-04: Program Execution

## Purpose

This document defines operational aspects of executing programs within SCP.

## In Scope / Out of Scope

**In Scope**
- Program rollout
- Execution monitoring
- Operational reporting

**Out of Scope**
- Strategic program design
- Financial disbursement execution

## Definitions

- Program Execution: Operational delivery of a program.
- Execution Status: Current operational state.

## Rules (MUST/SHOULD/MAY)

- Programs MUST be executed within approved scope.
- Execution MUST follow governance workflows.
- Progress SHOULD be tracked via defined KPIs.
- Baseline measurements MUST be captured before program interventions begin.
- Outcome measurements MUST be captured at defined checkpoints and at program closeout.
- Execution MAY be restricted or suspended by governance decisions.
- Automated alerts MUST be generated for execution delays.
- Emergency program changes MUST follow the normal governance workflow.

## Edge Cases

- Program execution during policy changes.
- Resource shortages impacting delivery.

## Audit & Logs

- Execution milestones MUST be logged.
- Deviations from plan MUST be auditable.
