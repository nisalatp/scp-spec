# SCP Specification Repository

This repository contains the Markdown specifications for the Smart Community Platform (SCP). Each numbered folder corresponds to a major area of the system.

## How to use

- Start with `00-index.md` for the complete table of contents.
- Each spec file follows a consistent structure and uses MUST/SHOULD/MAY for normative requirements.
- File names are stable references for cross-linking between sections.

## Structure

- 01-vision: Problem definition, goals, stakeholders, and success metrics.
- 02-architecture: System context, logical and physical architecture.
- 03-domain: Domain model and core entities.
- 04-security: Authentication, authorization, consent, and audit.
- 05-governance: Governance models and conflict resolution.
- 06-platform-core: Core registries, forms, and configuration.
- 07-verticals: System and custom vertical governance.
- 08-operations: Operational workflows and reporting.
- 09-integrations: Adapters, pipelines, and external integrations.
- 10-data: Data architecture, retention, and backup.
- 11-apis: API standards and service interfaces.
- 12-deployment: Kubernetes, CI/CD, secrets, and observability.
- 13-migration: Migration strategy and cutover.
- 14-testing: Test strategy and validation plans.
- 15-runbooks: Operational runbooks.
- 16-appendix: Glossary, samples, and templates.
