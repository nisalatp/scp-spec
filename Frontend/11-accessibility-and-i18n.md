# 11 Accessibility and Localization

This section covers accessibility and localization requirements.

## 1. Accessibility

- Use semantic HTML.
- Ensure keyboard navigation works.
- Provide labels for all inputs.
- Ensure sufficient color contrast.

## 2. Localization

- Use translation keys for all UI text.
- Support tenant specific languages.
- Dates and numbers must respect locale formats.

## Checklist

- Keyboard navigation works
- All inputs have labels
- UI strings are localizable
