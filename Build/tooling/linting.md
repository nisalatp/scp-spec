# Linting & Quality Standards

To ensure a clean codebase, all contributions MUST pass the following checks.

## Frontend (scp-frontend)
- **ESLint**: Standard JS/TS rules + React Hooks plugin.
- **SonarLint**: Security and code smell detection.

## Backend (scp-backend)
- **EditorConfig**: Uniform whitespace and encoding.
- **Static Analysis**: Specialized for .NET/Node.js to catch SQLi and performance anti-patterns.

## Documentation
- **Markdownlint**: To maintain consistency in documentation artifacts.
