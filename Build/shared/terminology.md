# Canonical Terminology

To avoid implementation drift, the following terms MUST be used consistently in code (classes, variables) and UI labels.

| Term | Implementation Keyword | Definition |
| :--- | :--- | :--- |
| **Beneficiary** | `Person` | An individual receiving or eligible for services. |
| **Household** | `Household` | A primary social unit residing at a single geo-point. |
| **Vertical** | `Vertical` | A high-level domain of governance (e.g., Health, Education). |
| **Geo Node** | `GeoNode` | An administrative or spatial unit in the geographic tree. |
| **Intervention**| `Intervention` | A specific action or item delivered as part of a Case. |
| **Case** | `Case` | The lifecycle of a specific program enrollment for a person. |
| **Audit Log** | `AuditLog` | The tamper-evident record of all system state changes. |
| **Consent** | `Consent` | The formal agreement by a citizen for their data usage. |
